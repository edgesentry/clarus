# clarus

Physics-based safety monitoring and compliance evidence platform. The same engine applies to any domain — the active use case is determined by the loaded profile.

**[API Reference](https://edgesentry.github.io/clarus/api-reference/) · [Interactive API Docs](https://clarus.edgesentry.io/api-docs) · [Docs](https://edgesentry.github.io/clarus/)**

| Profile | Domain | Primary submission |
|---|---|---|
| `sg-port-safety/` | Port worker / forklift safety — MPA + MOM WSH | PIER71-14 |
| `sg-maritime-security/` | Vessel restricted zone + AIS gap monitoring | CAP Vista (dual-use) |
| `sg-bca-greenmark/` | BCA Green Mark — EUI / COP / LPD thresholds | BEAMP (next cycle) |
| `sg-ot-cybersecurity/` | OT software integrity — component allowlist (IACS UR E26/E27) | PIER71-02 |
| `demo/` | Generic citations — OSS demo only | — |

---

## Scope

```
edgesentry-rs   Domain-agnostic Rust library + eds CLI.
                ZKP primitives (eds-zkp), audit chain, evaluate, profile engine.
                No business logic. Consumed by clarus via Cargo path deps.

clarus          Data acquisition, storage, and verification layer.
  edge/         Edge daemon — sensor ingestion → evaluate → sign → WORM upload + ZKP proof generation.
  analytics/    Cloudflare Pages — operations monitor, audit chain viewer, B2B verify API.
```

clarus owns all data generated by the edge (WORM chain, R2 buckets). Third parties verify compliance claims through the B2B API — they never access raw sensor data.

## Architecture

```
edgesentry-rs (engine + CLI)        clarus (this repo — app layer)
────────────────────────────        ──────────────────────────────
eds-zkp  ZkProgram / ZkProof        edge/  ZkProgram impl per profile
eds audit sign / verify                    → proof_bytes in WORM record
eds ingest / compute / evaluate            → R2 WORM bucket
eds assess / explain / report
                                    analytics/  Cloudflare Pages
                                      /api/verify      ← B2B verifier
                                      /api/verify/raw  ← raw proof for client-side verify
                                      /api/audit-*     ← chain inspection
                                      /live            ← operations monitor
                                      /api-docs        ← OpenAPI spec
```

**ZKP flow:**

```
edge daemon
  │ evaluate(profile, sensor data)
  │ → RiskEvent + GreenMarkAttestation (private: EUI/COP/LPD, public: cert_level)
  │ → ZkProof { proof_bytes = BLAKE3(public_values) }   [mock; SP1 pending]
  └─▶ WORM record  →  R2 audit bucket
              ↓
  /api/verify?site=xxx   ← BCA auditor, bank, tenant verifies here
  /api/verify/raw        ← B2B system integration
```

## Platform

clarus and [documaris](https://github.com/edgesentry/documaris) form the EdgeSentry platform:

```
clarus (monitoring + verification)     documaris (operator UI + compliance docs)
──────────────────────────────         ──────────────────────────────────────────
Edge daemon, WORM chain, R2            Reads attested data from clarus (browser)
/api/verify for 3rd parties            Generates BCA reports, FAL forms
https://clarus.edgesentry.io           https://documaris.edgesentry.io
```

Generated compliance documents embed a `verify_url` → `clarus.edgesentry.io/api/verify` so recipients can independently verify without going through documaris.

## Quick start

```bash
# 1. Build eds engine (sibling checkout required)
cd ../edgesentry-rs && cargo build -p eds

# 2. Run web app (analytics + API)
cd ../clarus/analytics && npm install && npm run dev
# → http://localhost:5173/api-docs  (API docs)
# → http://localhost:5173/live      (operations monitor)

# 3. Run desktop GUI
cd ../clarus/ui && npm install && npm run tauri dev
```
